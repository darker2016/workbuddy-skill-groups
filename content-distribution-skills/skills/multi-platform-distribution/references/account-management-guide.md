## 二、OAuth 认证流程

### 各平台认证机制对比
| 平台 | Token有效期 | 刷新机制 |
|------|-----------|---------|
| YouTube | 1小时 | Refresh Token |
| TikTok | 24小时 | Refresh Token |
| Instagram | 60天 | Long-lived Token |
| Facebook | 60天 | Long-lived Token |
| X (Twitter) | 2小时 | Refresh Token |
| 快手 | 可配置 | Refresh Token |
| B站 | 30天 | 重新授权 |
| 微信公众号 | 2小时 | Refresh Token |

### Token 生命周期管理
1. 初次授权：跳转平台授权页 → 确认权限 → 回调获取 Token
2. Token 刷新：定时检查 → 即将过期 → 自动续期
3. 认证失效：检测到401错误 → 自动刷新 → 失败则通知用户
4. 权限升级：需要新权限 → 引导用户重新授权

## 三、矩阵号管理策略

### 金字塔模型
```
        品牌主号 (1个)  ← 官方形象、品牌声量
           ↓
      达人号 (3-5个)  ← 垂直品类、人设化
           ↓
    素人号矩阵 (10-50个)  ← 铺量、SEO
```

### 各层级职责
| 层级 | 定位 | 发布频率 |
|------|------|---------|
| 品牌主号 | 官方权威 | 3-5条/周 |
| 达人号 | 人设种草 | 1-2条/日 |
| 素人号 | 铺量覆盖 | 1条/日 |

### 矩阵号协同策略
1. 内容差异化：同一主题不同角度（≥30%差异）
2. 发布时间错开：各号间隔≥2小时
3. 互动协同：子号在主号评论区自然互动
4. 流量引导：子号bio/评论引导至主号
5. 账号隔离：不同设备/IP/指纹

## 四、设备指纹防关联

### 核心原则
- **硬件隔离**：一机一号，独立SIM卡，不同WiFi/移动数据
- **软件隔离**：独立浏览器指纹，隔离Cookie
- **IP管理**：每个账号绑定独立IP段，地理位置与注册地匹配

### 防关联操作规范
1. 注册阶段：不同设备/IP/手机号，间隔>24h
2. 养号阶段：前7天仅浏览、点赞，不发布
3. 运营阶段：固定设备+IP操作，不混用
4. 换设备：需间隔>7天
5. 紧急情况：一个号被封不连坐其他号

## 五、封号风险规避

### 通用红线
| 风险等级 | 行为 | 后果 |
|---------|------|------|
| 🔴 高危 | 批量搬运他人内容 | 封号 |
| 🔴 高危 | 违法违规内容 | 封号+追责 |
| 🔴 高危 | 外挂/作弊工具 | 封号 |
| 🟡 中危 | 频繁操作 | 限流/临时封禁 |
| 🟡 中危 | 硬广无标识 | 限流/内容删除 |
| 🟢 低危 | 发布间隔过密 | 限流 |

### 各平台特有红线
**抖音**：搬运检测极严，DOU+投放不能引导加微信
**小红书**：品牌合作必须报备，禁止极限词，图片不能有其它APP水印
**B站**：搬运需标注来源，恰饭需标"包含推广"
**YouTube**：音乐版权检测（Content ID），误导性缩略图会被处罚
**TikTok**：各地区审核标准不同，音乐版权因地区而异

## 六、账号健康度评估

### 健康度评分模型
```
账号健康分 = 认证状态(20%) + 活跃度(20%) + 内容质量(25%) + 互动质量(20%) + 合规记录(15%)
评级：90-100 优秀 | 70-89 良好 | 50-69 一般 | <50 风险
```

### 日常巡检清单
- [ ] 所有账号Token状态正常
- [ ] 无平台通知/警告
- [ ] 近24h发布内容状态正常
- [ ] 互动数据无异常波动
- [ ] 粉丝数无异常下降
- [ ] API调用配额充足

## 七、新号养号 SOP

| 阶段 | 时间 | 操作 |
|------|------|------|
| 注册期 | Day 1 | 完善所有资料、绑定手机/邮箱 |
| 浏览期 | Day 1-3 | 每天浏览30-60分钟，点赞5-10条 |
| 互动期 | Day 4-7 | 开始评论、收藏，增加互动频率 |
| 试发期 | Day 8-14 | 发布1-2条测试内容，观察数据 |
| 正式期 | Day 15+ | 按正式节奏运营 |

**养号期注意事项**：
- 前7天绝对不发布内容，只浏览和互动
- 行为要像真实用户（有浏览、有停留、有互动）
- 不要一注册就进行任何商业行为
- 每日操作时长保持在30-60分钟
