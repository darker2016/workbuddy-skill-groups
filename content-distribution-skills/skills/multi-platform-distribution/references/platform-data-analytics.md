# 多平台数据分析框架

## 一、数据聚合架构

```
原始数据层（各平台 API 返回）
    ↓ 数据清洗 + 口径统一
标准化数据层（统一指标定义）
    ↓ 聚合计算
分析层（跨平台对比 + 趋势分析）
    ↓ 洞察提炼
决策层（优化建议 + 行动方案）
```

## 二、核心指标体系

### 通用指标
| 指标 | 统一定义 |
|------|---------|
| 曝光量 | 内容被展示的总次数 |
| 触达人数 | 看到内容的独立用户数 |
| 播放量 | 视频被播放的次数 |
| 完播率 | 完播次数 / 播放次数 × 100% |

### 互动指标权重
| 指标 | 权重 |
|------|------|
| 点赞数 | 1× |
| 评论数 | 3× |
| 分享数 | 5× |
| 收藏数 | 2× |
| 关注转化 | 10× |

### 综合计算指标

**标准互动率**：
```
互动率 = (点赞 + 评论 + 分享 + 收藏) / 曝光量 × 100%
```

**加权互动率**：
```
加权互动率 = (点赞×1 + 评论×3 + 分享×5 + 收藏×2) / 曝光量 × 100%
```

**内容质量分（0-100）**：
```
CQS = 互动率得分(40%) + 完播率得分(30%) + 粉丝转化得分(20%) + 分享率得分(10%)
各项得分 = min(100, (实际值 / 平台基准值) × 60)
```

## 四、数据归一化方法

### 方案一：百分位排名法
将每个平台的数据换算为同平台同量级账号中的百分位排名。

### 方案二：基准比值法
```
归一化值 = 实际值 / 平台基准值
示例：抖音互动率 8%，基准 5% → 归一化 = 1.6（超标60%）
```

### 方案三：Z-Score标准化
```
Z = (X - μ) / σ
Z > 1.5：爆款级别
Z 0.5-1.5：良好表现
Z -0.5~0.5：正常范围
Z < -0.5：需关注
Z < -1.5：需诊断
```

## 五、效果评估模型

### 分发效率维度
| 维度 | 计算 | 意义 |
|------|------|------|
| 覆盖效率 | 总曝光 ÷ 总发布数 | 单条平均曝光 |
| 互动效率 | 总互动 ÷ 总发布数 | 单条平均互动 |
| 转化效率 | 新增粉丝 ÷ 总发布数 | 单条平均获粉 |
| 平台ROI | 效果总值 ÷ 投入时间 | 投入产出比 |

## 六、A/B 测试方法

### 可测试变量
| 变量 | 样本量要求 |
|------|-----------|
| 发布时间 | ≥10对 |
| 封面设计 | ≥5对 |
| 标题文案 | ≥10对 |
| 内容长度 | ≥5对 |
| 标签策略 | ≥10对 |
| 发布频率 | ≥4周 |
| 内容格式 | ≥5对 |

### 测试设计原则
1. 单变量控制：每次只改变一个变量
2. 样本量充足：避免偶然性
3. 时间对齐：对比组在相同时间段内执行
4. 统计显著性：差异超过20%且持续3次以上才确认
5. 迭代验证：初步结论需在下一轮重复验证
