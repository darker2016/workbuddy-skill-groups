# QA Issue Taxonomy — Issue 分类学

> QA 测试中的 Bug 分类标准。定义严重度分级、分类体系和逐页检查清单。
> 本文件是 `08-qa-skill.md` 的引用资源。

---

## 严重度分级

| 严重度 | 定义 | 举例 |
|--------|------|------|
| **critical** | 阻塞核心工作流，导致数据丢失或应用崩溃 | 表单提交导致错误页面、结账流程断裂、数据未确认即删除 |
| **high** | 主要功能不可用或无法正常使用，无替代方案 | 搜索返回错误结果、文件上传静默失败、认证重定向循环 |
| **medium** | 功能可用但有明显问题，有替代方案 | 页面加载慢（>5s）、表单验证缺失但提交仍能成功、仅移动端布局断裂 |
| **low** | 轻微外观或修饰问题 | 页脚拼写错误、1px 对齐问题、悬停状态不一致 |

## 分类

### 1. Visual/UI（视觉/UI 问题）
- 布局断裂（元素重叠、文本被裁剪、水平滚动条）
- 图片缺失或破损
- z-index 错误（元素出现在其他元素后面）
- 字体/颜色不一致
- 动画异常（卡顿、过渡不完整）
- 对齐问题（偏离网格、间距不均匀）
- 深色模式/主题问题

### 2. Functional（功能问题）
- 链接断裂（404、错误目的地）
- 按钮无效（点击无反应）
- 表单验证（缺失、错误、可绕过）
- 重定向错误
- 状态不持久（刷新后数据丢失、返回按钮异常）
- 竞态条件（重复提交、陈旧数据）
- 搜索返回错误或无结果

### 3. UX（用户体验问题）
- 导航混乱（无面包屑、死胡同）
- 缺少加载指示器（用户不知道正在发生什么）
- 交互响应慢（>500ms 无反馈）
- 错误信息不清晰（"出了点问题"无进一步详情）
- 破坏性操作前无确认
- 跨页面交互模式不一致
- 死胡同（无返回路径、无下一步操作）

### 4. Content（内容问题）
- 拼写和语法错误
- 过时或不正确的文本
- 遗留的占位文本 / lorem ipsum
- 文本被截断（无省略号或"更多"）
- 按钮或表单字段标签错误
- 空态缺失或无用

### 5. Performance（性能问题）
- 页面加载慢（>3 秒）
- 滚动卡顿（丢帧）
- 布局偏移（加载后内容跳跃）
- 页面网络请求过多（>50）
- 大型未优化的图片
- 阻塞 JavaScript（加载期间页面无响应）

### 6. Console/Errors（控制台/错误）
- JavaScript 异常（未捕获的错误）
- 网络请求失败（4xx, 5xx）
- 弃用警告（即将断裂）
- CORS 错误
- 混合内容警告（HTTPS 页面中的 HTTP 资源）
- CSP 违规

### 7. Accessibility（可访问性问题）
- 图片缺少 alt 文本
- 表单输入无标签
- 键盘导航损坏（无法 Tab 到元素）
- 焦点陷阱（无法逃离弹窗或下拉框）
- 缺失或不正确的 ARIA 属性
- 颜色对比度不足
- 屏幕阅读器无法访问的内容

## 逐页探索清单

QA 会话中每访问一个页面时：

| # | 步骤 | 说明 |
|---|------|------|
| 1 | **Visual scan** | 视觉扫描：布局问题、图片破损、对齐 |
| 2 | **Interactive elements** | 点击每个按钮、链接、控件，验证是否正常工作 |
| 3 | **Forms** | 填写并提交，测试空提交、无效数据、边界情况 |
| 4 | **Navigation** | 检查所有进出路径：面包屑、返回按钮、深度链接、移动菜单 |
| 5 | **States** | 检查空态、加载态、错误态、溢出态 |
| 6 | **Console** | 交互后检查控制台有无新 JS 错误或失败请求 |
| 7 | **Responsiveness** | 检查移动端和平板视口 |
| 8 | **Auth boundaries** | 登出后看到什么？不同角色看到什么？ |
