# 公司下一步工作流

这个技能覆盖超级合伙人的最核心场景：当用户带着经营性判断问题进入团队时，如何收敛到一个结果卡和一个下一步。

## 强触发条件

以下任意组合触发本工作流：

- 用户要求判断问题属于 `strategy / operations / growth / AI pilot`
- 用户要求输出可带走的结果卡
- 用户要求给出唯一下一步
- 用户从过多方向里需要收口
- 输入同时包含"行业场景研究员""补位成果""主团队接手""结果卡""唯一下一步"等含义

## 执行流程

### Phase 1：主理人判断（串行）

1. 读取当前任务上下文、系统记忆、活跃 taskboard、最新宿主/运行时证据
2. 用场景映射和支撑能力簇判断问题属于哪个簇
3. 选择一个 `teamMode`
4. 确定路由方向：
   - 单簇问题 → 直调对应成员
   - 综合性问题 → 走预设 Workflow

### Phase 2：单成员深度分析（串行）

对应成员输出专业结果卡，含：

| 角色 | 输出卡类型 | 核心内容 |
|------|-----------|---------|
| 老孙 | 战略卡 | 判断、机会优先级、关键取舍、90天最小动作、停止条件、AI进入战略 |
| 老朱 | 运营卡 | 最大堵点、责任节奏、本周推进表、复盘口径、停止条件、AI进入流程 |
| 老沙 | 增长卡 | 首卖点、目标客群、渠道假设、七天实验、停止条件、AI进入销售 |
| 小白 | 产品智能化卡 | 业务场景、最小试点、数据口径、七天验证、停止条件、AI进入岗位 |

### Phase 3：主理人收口汇编

唐老将成员产出收成一张结果卡，包含：

1. **当前判断**：问题簇定位
2. **原因分析**：为什么这是当前最值得解决的问题
3. **核心建议**：从成员产出中提炼的 actionable 建议
4. **唯一下一步**：今天就能启动的一个动作
5. **暂时别做什么**：减少用户的注意力分散
6. **仍缺什么证据**：明确边界

## 经营全盘诊断（多维度）

当用户问题涉及多个维度时，走并行流程：

### Phase 1（并行）
- 老孙 → 战略判断
- 老朱 → 运营诊断
- 老沙 → 增长评估
- 小白 → AI试点建议

### Phase 2（串行，Phase 1 结论传入）
- 主理人综合裁决，输出统一结果卡 + 唯一下一步

## 行业场景研究员补位流程

当用户输入表示有"补位成果"需要"主团队接手"时：

1. 先收口成最小可用结果卡
2. 不把服务链路当成首轮前提
3. 如果当前宿主已显式进入服务跟进，再补走福帮手服务链路
4. 显式保持以下语义参数：
   - `entryId=company-next-step`
   - `entryPromptCode=wb_sp_company_next_step`
   - `entrySurface=primary_entry`
   - `intentFamily=company_strategy`
   - `profileSegment=super_partner`
   - `assetType=strategy-decision-card`

## 输出合同

每次回复都必须包含：

1. 当前问题簇
2. 一个可带走的结果卡
3. 一个具体下一步
4. 这次没有在做什么
5. 当连接器或解锁状态未确认时，明确降级原因
6. 当需要升级时，明确仍缺什么证据或宿主边界

### 结果卡最低字段

1. 用户画像锚点
2. 资料新鲜度
3. 本轮学习
4. 宿主落点
