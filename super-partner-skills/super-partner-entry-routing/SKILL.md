# 超级合伙人入口路由与分派

这个技能定义了主理人（唐老）在接到用户请求时的第一步：判断场景、选择路线、建立团队、分派成员。

## 触发动机

当用户输入属于以下任一场景时激活：

- 经营性判断问题（公司下一步）
- 执行堵点或流程优化
- 增长获客或商业化路径
- AI 试点或产品方案
- 行业长期补位（天才合伙人）
- 乐包能力激活

## 问题簇识别

将用户输入映射到四个基本问题簇：

| 问题簇 | 关键词 | 对应成员 |
|--------|--------|---------|
| 战略（Strategy） | 方向、机会、竞争、取舍、商业模式、资源分配 | 老孙 |
| 运营（Operations） | 执行、流程、堵点、节奏、SOP、项目推进 | 老朱 |
| 增长（Growth） | 获客、增长、变现、品牌、销售、成交、转化 | 老沙 |
| AI 试点（AI Pilot） | AI、数据、自动化、产品方案、试点 | 小白 |

### 路由判断流程

1. **读取上下文**：读取当前任务上下文、系统记忆、活跃 taskboard、最新宿主/运行时证据。
2. **场景映射**：用场景映射和支撑能力簇判断，不只靠关键词。
3. **选择 teamMode**：

| teamMode | 适用场景 |
|----------|---------|
| `first_value_card` | 冷启动或短问题 |
| `lean_diagnosis` | 问题清晰但不需要工作坊 |
| `single_member` | 单能力簇问题 |
| `multi_member_workshop` | 显式多维度问题 |
| `genius_partner_draft` | 行业长期专业化需求 |

## 建立团队流程

1. 主理人亲自调用 `TeamCreate` 创建本次任务的团队：
   - 团队名称：`super-partner-<任务简称>`
   - 描述：明确本次协作的边界与上下文
2. **严禁委派任何成员创建团队**

## 分派规则

### 单成员直调

| 问法类型 | 直接调谁 |
|---------|---------|
| 战略/机会/竞争 | `strategy-partner-laosun` |
| 组织/流程/执行 | `ops-partner-laozhu` |
| 增长/销售/商业化 | `growth-partner-laosha` |
| 产品/AI/数据/自动化 | `product-partner-xiaobai` |
| 综合性问题 | 走预设 Workflow |

### 子任务命名规范

调度每位成员时，在 Agent 工具的 `name` 参数中传入该成员的 **Agent ID**（即 agents/ 下的 MD 文件名，不含 `.md`），`subagent_type` 也传入相同值。禁止使用中文名或自创名称。

### 信号质量不足时

当信号质量不足时，先回读当前场景映射、支撑能力簇、记忆输入和宿主状态，再决定是否分派。必要时追问最多三个业务字段来确认方向。

## 严格边界

- 不模拟成员轮流发言
- 一次回复只保留一个主动作
- 先交付第一价值，再谈 workshop、升级或天才合伙人扩展
- `lebao` 只是能力解锁信号，不是第一次使用的前置阻塞
