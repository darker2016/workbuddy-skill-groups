# 乐包激活与能力继续

乐包（lebao）是能力解锁信号，表示用户已获得某项新能力或新工具的访问权限。这个技能定义了如何将乐包信号转化为可执行的行动。

## 乐包是什么

- **lebao = 能力解锁信号**，不是特权或营销概念
- 表示用户已经跨过一个能力门槛
- 可能是工具、知识库、专家包、连接器或其他能力的解锁

## 核心原则

1. `lebao` 是能力解锁信号，不是第一次使用的前置阻塞
2. 先交付第一价值，再谈乐包激活后的进阶功能
3. 乐包激活本身不是结果，后续行动才是

## 执行流程

### Step 1：确认解锁状态

检查当前乐包解锁状态：

| 状态 | 含义 | 处理方式 |
|------|------|---------|
| `locked` | 未解锁 | 按正常首值流程处理，不提及乐包 |
| `preview` | 预览状态 | 可展示能力范围，但部分功能受限 |
| `unlocked` | 已解锁 | 进入 Step 2 激活流程 |
| `expired` | 已过期 | 提示续期或降级使用 |

### Step 2：识别解锁能力

确定解锁了什么：

- 是哪个成员/能力簇的升级？
- 是新的工具/连接器可用？
- 是场景包/SOP 的扩展？
- 是新数据源的接入？

### Step 3：产出继续行动卡

基于解锁能力，生成一张能力继续卡：

1. **解锁摘要**：解锁了什么能力，当前状态
2. **能力简介**：用业务语言说明这能做什么
3. **推荐使用场景**：3 个以内的典型使用场景
4. **第一个动作**：今天就能试的一个使用动作
5. **与当前工作的衔接**：如何与正在推进的业务结合
6. **下一步触发**：什么信号意味着应该继续深入

### Step 4：保持路线语义

```
entryId=company-next-step      # 或对应解锁前的 entryId
entrySurface=extended_entry    # 乐包激活是扩展入口
assetType=lebao-unlock-card
```

## 乐包激活后的分支路径

### 路径 A：能力继续（默认）
解锁了一个能力的增强版本 → 继续在原有路线上深化。

### 路径 B：新能力接入
解锁了一个全新的能力 → 安排首次使用的教程/指引。

### 路径 C：主团队汇聚
如果乐包激活来自"行业场景研究员补位" → 进入主团队接手流程，将补位成果收成结果卡。

## 与其他技能的衔接

- 如果激活后需要服务跟进 → 转 `super-partner-fbs-followthrough`
- 如果激活后需要深入行业补位 → 转 `super-partner-genius-draft`
- 如果只是能力升级 → 保持在当前路线继续

## 硬规则

1. 不要把乐包包装成特权或稀缺资源
2. 不要因为乐包解锁了就跳到超出用户当前问题范围的扩展内容
3. 乐包激活后的第一个动作必须用户今天就能做
4. 如果乐包状态无法确认，按 `locked` 处理，不要猜测
