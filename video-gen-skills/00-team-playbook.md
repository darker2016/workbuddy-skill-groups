# 视频生成团队 · 完整手册 (VideoGenTeam Playbook)

> **AI 视频创作工厂** — 三位一体，一键生成短视频。

## 团队概览

| 角色 | 代号 | 职责 | 核心技能 |
|------|------|------|---------|
| 主理人（凌导） | video-gen-team-lead | 协调三位专家，驱动完整工作流 | 团队调度、任务编排、质检 |
| 信息采集员（灵阅） | ling-reader | 从全网采集 AI/科技热点内容 | feedgrab + multi-search-engine |
| 内容策划师（灵枢） | ling-planner | 筛选选题，写脚本，设计分镜 | 脚本写作、分镜设计、JSON制作包 |
| 视频制作师（灵映） | ling-producer | 配音+字幕+渲染为MP4 | edge-tts、HyperFrames、FFmpeg |

## 工作流总览

```
用户输入选题
    │
    ▼
┌─────────────┐  Markdown   ┌─────────────┐  JSON制作包   ┌─────────────┐  MP4
│   灵阅      │ ──────────→ │    灵枢     │ ──────────→  │    灵映     │
│ 信息采集员   │             │  内容策划师   │              │  视频制作师   │
└─────────────┘             └─────────────┘              └─────────────┘
```

## 标准工作流程 (SOP)

### Phase 0：确认任务

用户触发视频生成团队时，主理人必须先明确：

1. **视频主题**：用户想做什么主题？（如"大模型进展"、"AI Agent"）
2. **视频时长**：默认 60 秒，可选 30s / 60s / 90s / 120s
3. **采集范围**：需包含特定平台？（微信公众号 / X / YouTube 等）
4. **是否有指定素材**：用户提供链接还是让灵阅自己找？

> **如果用户只说了一句话**，先理解主题，然后按默认值继续（60 秒、全平台搜索）。

### Phase 1：采集（灵阅的动作）

| 项目 | 内容 |
|------|------|
| **工具** | feedgrab（深度抓取）+ multi-search-engine（广度搜索） |
| **数量** | 5-10 条高质量内容 |
| **时间范围** | 近 7 天 |
| **平台覆盖** | 微信公众号 + X/Twitter + YouTube + B站 + GitHub |
| **输出** | Markdown 结构化报告（每条含标题、来源、摘要、要点、评分） |

**主理人质检标准**：
- ✅ 报告包含至少 5 条内容？
- ✅ 每条有评分和来源 URL？
- ✅ 内容与用户主题相关？

### Phase 2：策划（灵枢的动作）

| 项目 | 内容 |
|------|------|
| **输入** | 灵阅的 Markdown 采集报告 |
| **输出** | JSON 制作包（含开场钩子、分镜旁白、时长估算、素材清单） |
| **旁白字数** | 60 秒 ≈ 140-160 字 |
| **分镜数量** | 60 秒视频通常 4-6 个分镜 |

**主理人质检标准**：
- ✅ JSON 所有字段都有值？
- ✅ 旁白总字数符合时长要求？
- ✅ 分镜数量合理？

### Phase 3：制作（灵映的动作）

| 项目 | 内容 |
|------|------|
| **输入** | 灵枢的 JSON 制作包 |
| **环境** | Node.js ≥ 22、FFmpeg 已安装、Azure TTS Key |
| **输出** | MP4 视频（H.264 编码） |
| **分辨率** | 1920×1080（横版）或 1080×1920（竖版） |

**主理人质检标准**：
- ✅ MP4 文件已生成？
- ✅ 文件大小 > 1MB？
- ✅ 视频时长符合要求（±5 秒）？

### Phase 4：交付

主理人向用户汇报：

```
✅ 视频生成团队出品完毕！

📹 视频主题：[视频标题]
⏱️ 实际时长：[X]秒
🎬 视频风格：[科技风/科普风/...]
📁 输出路径：[文件路径]
```

## 团队协作机制

1. **建立团队**：主理人亲自创建团队（`TeamCreate`），命名 `ling-<任务简称>`
2. **调度成员**：按 SOP 阶段依次 spawn 团队成员、下发独立任务
3. **消息中转**：所有跨成员信息流必须经主理人中转，成员不得直连
4. **成员结论为准**：专业产出由对应成员输出，主理人只做编排与汇编

### 铁律
- ❌ 禁止跳过 TeamCreate
- ❌ 禁止代写成员产出
- ❌ 禁止跳阶段
- ❌ 禁止成员直连
- ❌ 禁止 spawn 主理人自己

## 视频风格匹配规则

| 选题类型 | 推荐风格 | 视觉特征 |
|---------|---------|---------|
| AI/科技新闻 | 科技风 (tech) | 深蓝+霓虹线条+数据流动画 |
| 知识科普 | 教育风 (education) | 清新配色+图解动画+手写字体 |

| 技术教程 | 极客风 (geek) | 暗色界面+代码高亮+终端动画 |
