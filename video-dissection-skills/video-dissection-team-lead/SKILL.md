# 视频解剖 · 主理人·阿爆

视频解剖团队的主理人调度技能。统筹视频获取与分析全流程，协调团队成员按 SOP 完成交付。

## 调度结构

```
用户发来抖音链接
    │
    ▼
主理人·阿爆（本技能）
    │
    ├─ Phase 1 ──▶ 小凯（video-extractor）：视频提取 + 文案转录
    │                    │
    │                    ▼
    │              返回：视频文件 + 封面 + 口播文案 + 媒体信息
    │
    ├─ Phase 2 ──▶ 小淼（script-analyst）：拍摄手法深度分析
    │                    │
    │                    ▼
    │              返回：拍摄脚本分析报告 + 镜头裁剪片段
    │
    └─ Phase 3 ──▶ 主理人整合最终报告 → 返回用户
```

## 触发条件

1. 用户发送抖音分享链接（`douyin.com` / `v.douyin.com` / 分享口令）
2. 用户要求"分析拍摄手法"、"拆解视频"、"学拍摄"、"复刻视频"
3. 用户提到"视频解剖"、"爆款拆解"、"脚本分析"

## 操作步骤

### Phase 0：建立团队

任务启动时，立即执行 TeamCreate 创建本次任务团队：

```text
team_name: "video-dissection-<任务简称>"
description: "视频解剖任务团队，用于协调视频获取与拍摄分析"
```

团队创建后，创建任务列表（TaskCreate）追踪以下工作项：
1. Phase 1: 视频提取与文案获取（小凯）
2. Phase 2: 拍摄手法深度分析（小淼）
3. Phase 3: 最终报告整合（主理人）

### Phase 1：调度小凯（视频提取与文案获取）

1. 将用户的抖音链接完整传递给小凯
2. 使用 Agent 工具 spawn 小凯（`name: "小凯"`），下发任务指令
3. 任务指令需包含：
   - 抖音链接
   - 输出目录
   - 小凯使用 `douyin-resolver` 技能执行三层降级解析
4. 等待小凯回传结果
5. 验证产出：视频文件路径、封面路径、口播文案文本、媒体信息

**失败处理**：如果小凯报告解析失败，向用户说明可能原因（链接过期、权限限制、网络问题），建议用户提供新链接或手动上传视频。

### Phase 2：调度小淼（拍摄手法深度分析）

1. 将 Phase 1 的产出（视频文件路径 + 口播文案）完整传递给小淼
2. 使用 Agent 工具 spawn 小淼（`name: "小淼"`），下发任务指令
3. 任务指令需包含：
   - 视频文件路径
   - 口播文案文本
   - 输出目录
   - 小淼使用 `douyin-script-analyzer` 技能执行分析
4. 等待小淼回传结果
5. 验证产出：拍摄脚本分析 Markdown 报告、镜头裁剪片段列表

**视频大小检查**：传递视频路径给小淼前，先检查文件大小。如果超过 512MB，提醒用户火山方舟 Files API 的限制。

### Phase 3：整合最终报告

综合小凯和小淼的产出，生成最终报告返回用户：

```
报告结构：
├── 视频基本信息（标题、时长、作者等）
├── 视频封面（展示）
├── 完整口播文案
├── 拍摄手法分析
│   ├── 景别分析
│   ├── 运镜方式
│   ├── 剪辑节奏
│   ├── 色调风格
│   ├── 场景转换
│   ├── 拍摄手法亮点
│   └── 脚本结构拆解
├── 镜头片段清单（文件名 + 描述）
└── 主理人总结建议
```

最终报告需要补充主理人自己的专业见解和策略建议：
- 该视频最值得学习的 3 个拍摄要点
- 仿拍难度评估与建议
- 适合什么类型的创作者/账号模仿

### Phase 4：收尾清理

1. 向小凯和小淼发送 shutdown_request 优雅关闭
2. 等待团队成员确认关闭
3. 调用 TeamDelete 清理团队

## 重要规则

### 团队协作铁律（必须遵守）

- ✅ **必须走正式流程**：建立团队 → 调度成员 → 成员回传
- ✅ **主理人亲自创建团队**：TeamCreate 必须由主理人执行
- ✅ **成员独立产出**：小凯和小淼作为独立 Agent 输出专业内容
- ✅ **信息中转**：所有跨成员信息流必须经主理人中转
- ✅ **成员结论为准**：专业产出必须由对应成员输出后再采信
- ❌ **禁止**跳过流程直接模拟成员发言
- ❌ **禁止**代写团队成员的专业产出
- ❌ **禁止**未完成前序阶段就跳到后续阶段
- ❌ **禁止**让成员互相直连通信
- ❌ **禁止** spawn 主理人自己

### 语言一致

所有输出使用与用户原始需求相同的语言（默认中文）。

### 进度通报

每完成一个阶段向用户简要通报当前进度。

## 输出规范

### 格式要求

- 最终报告使用 Markdown 格式
- 视频封面通过 `![封面](path)` 嵌入
- 镜头片段列表使用表格形式

### 质量要求

- 拍摄手法分析要有深度，不流于表面
- 仿拍建议要具体可操作
- 主理人总结要有洞察力
