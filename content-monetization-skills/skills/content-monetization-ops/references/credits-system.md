# 积分/信用体系设计蓝图

## 体系概述

积分/信用体系是内容商业化平台的核心经济引擎，通过虚拟货币（积分）和信誉评价（信用分）双轨制度，激励优质行为、惩罚违规行为、促进交易流动。

## 一、积分体系设计

### 1.1 积分获取渠道

| 渠道 | 获取方式 | 积分数量 | 频率限制 | 设计意图 |
|------|---------|---------|---------|---------|
| 任务完成 | 按时高质量完成推广任务 | S级500/A级300/B级100 | 无上限 | 激励核心行为 |
| 内容上架 | 内容被采用到交易市场 | 50积分/条 | 日上限500 | 鼓励内容供给 |
| 交易佣金 | 撮合成功获得服务费 | 交易额×1% | 无上限 | 平台核心收入 |
| 活跃奖励 | 每日签到+连续签到 | 日10/周100/月500 | 每日一次 | 提高留存 |
| 优质内容 | 获评优质/精选内容 | 200积分/条 | 日上限3条 | 鼓励质量 |
| 邀请奖励 | 成功邀请新用户注册 | 100积分/人 | 日上限10人 | 拉新获客 |
| 充值购买 | 直接充值购买积分 | 1元=100积分 | 无上限 | 现金流入 |
| 特殊活动 | 节假日/大促活动奖励 | 双倍积分 | 活动期限内 | 短期刺激 |

### 1.2 积分消耗场景

| 场景 | 消耗数量 | 场景说明 | 消耗占比 |
|------|---------|---------|---------|
| AI 视频生成 | 500积分/次 | 调用AI生成短视频 | 35% |
| AI 图片生成 | 100积分/次 | 调用AI生成图片 | 20% |
| 数据分析报告 | 200积分/份 | 获取深度数据报告 | 15% |
| 推广位购买 | 1000积分/天 | 首页/推荐位展示 | 10% |
| 加急服务 | 200积分/次 | 任务加急处理 | 8% |
| 高级功能 | 300积分/月 | 竞品监控等高级功能 | 7% |
| 提现手续费 | 提现额×2% | 积分提现为现金 | 5% |

### 1.3 积分经济学

**通胀控制机制**：
- 每月积分发放总量封顶：平台月活用户 × 平均获取量 × 1.2（弹性系数）
- 动态调节：当月积分流通量超标时，降低非核心渠道获取量
- 消耗引导：定期推出限时消耗活动（如积分翻倍兑换）

**过期机制**：
- 有效期：获取后 12 个月
- 提前提醒：到期前 30 天/7 天/1 天分别提醒
- 延期条件：当月有消费行为可自动延期 3 个月

**等级体系**：

| 等级 | 累计积分 | 权益 |
|------|---------|------|
| 铜牌 | 0-999 | 基础功能 |
| 银牌 | 1000-4999 | 优先推荐+5%佣金加成 |
| 金牌 | 5000-19999 | 专属客服+10%佣金加成+数据看板 |
| 钻石 | 20000+ | 独家任务+15%佣金加成+全功能解锁 |

**兑换比例**：
- 积分→现金：100积分 = 1元（提现时）
- 现金→积分：1元 = 100积分（充值时）
- 最低提现：5000积分（=50元）

### 1.4 事件驱动架构

```
用户行为事件
    │
    ├─ 购买事件(Purchase)
    │   → Purchase Consumer
    │   → 验证支付状态
    │   → 积分入账（实时）
    │   → 发送入账通知
    │
    ├─ 消费事件(Consume)
    │   → Service 层处理
    │   → 余额检查
    │   → 扣除积分
    │   → 记录消费明细
    │   → 更新等级状态
    │
    ├─ 退款事件(Refund)
    │   → Refund Consumer
    │   → 验证退款条件
    │   → 积分返还
    │   → 标记退款记录
    │
    └─ 过期事件(Expire)
        → 定时任务扫描
        → 标记过期积分
        → 发送过期通知
        → 更新可用余额
```

### 1.5 退款机制

| 退款场景 | 处理方式 | 时效 |
|---------|---------|------|
| 任务失败（系统原因） | 全额自动退还 | 实时 |
| 任务取消（创作者原因） | 全额退还-10%违约金 | 1工作日 |
| 任务取消（商家原因） | 全额退还给创作者 | 1工作日 |
| 质量争议 | 仲裁后按比例退还 | 5工作日 |
| 服务不满意 | 7天无理由退还 | 审核后1工作日 |
| 重复扣费 | 全额自动退还 | 实时 |

## 二、结算系统设计

### 2.1 结算周期

| 类型 | 周期 | 适用场景 | 最低结算额 |
|------|------|---------|-----------|
| 日结 | 每日结算 | 小额高频（日累计≥50元） | 50元 |
| 周结 | 每周一结算 | 标准创作者 | 100元 |
| 月结 | 每月1日结算 | 大客户/MCN | 500元 |
| 即时结算 | 任务完成即结 | VIP创作者特权 | 无门槛 |

### 2.2 对账流程（5步标准）

```
Step 1: 数据采集
  ├─ 从各平台 API 拉取交易/互动/曝光数据
  ├─ 内部任务系统获取任务完成数据
  └─ 汇总生成原始数据清单

Step 2: 规则计算
  ├─ 按 CPS 规则：成交金额 × 佣金比例
  ├─ 按 CPE 规则：有效互动量 × 互动单价
  ├─ 按 CPM 规则：有效曝光量 / 1000 × CPM单价
  └─ 扣除平台服务费（5-15%）

Step 3: 异常检测
  ├─ 单条记录偏差 >20% 自动标记
  ├─ 总额与预期偏差 >30% 触发人工审核
  ├─ 刷量/异常互动自动排除
  └─ 退货/取消订单自动扣除

Step 4: 商家确认
  ├─ 发送电子对账单给商家
  ├─ 7天异议期（工作日计）
  ├─ 逐条确认或提出异议
  └─ 超时未响应视为确认

Step 5: 创作者到账
  ├─ 确认无误后发起打款
  ├─ 银行转账：T+1到账
  ├─ 平台钱包：实时到账
  └─ 发送到账通知
```

### 2.3 争议处理机制

**争议流程**：
1. 商家/创作者在异议期内提出争议
2. 提交证据材料（截图/后台数据/合同条款）
3. 平台仲裁团队介入（2个工作日内受理）
4. 双方补充举证（各1次机会）
5. 仲裁裁定（受理后5个工作日内）
6. 执行裁定（裁定后1个工作日内）

**常见争议类型及处理**：

| 争议类型 | 处理原则 | 举证责任 |
|---------|---------|---------|
| 数据不一致 | 以平台后台数据为准 | 申诉方提供反证 |
| 互动质量争议 | 按防刷量规则重新计算 | 平台举证 |
| 退货扣佣 | 按约定退货窗口期执行 | 商家提供退货记录 |
| 创作质量争议 | 参照任务Brief标准 | 商家说明不达标点 |
| 时效争议 | 以系统时间戳为准 | 申诉方提供时间证据 |

### 2.4 税务合规

**国内税务处理**：
- 个人创作者：劳务报酬所得，平台代扣代缴
  - 每次收入≤800元：免税
  - 800-4000元：(收入-800) × 20%
  - >4000元：收入 × (1-20%) × 税率（20%/30%/40%）
- 企业/MCN：开具增值税发票，自行申报
- 年度汇算：次年3-6月汇算清缴

**国际税务处理**：
- 美国税务居民：需填写 W-9 表格，平台报送 1099
- 非美国税务居民：需填写 W-8BEN 表格
- 欧盟地区：VAT 增值税合规（数字服务税）
- 跨境结算：注意汇率变动和手续费

**发票与凭证**：
- 电子对账单：每次结算自动生成 PDF
- 年度收益汇总：年末自动生成（含税前/税后）
- 完税证明：可在线申请下载
- 合同备案：每次合作的电子合同存档
