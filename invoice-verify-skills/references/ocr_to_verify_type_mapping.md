# 发票名称 → 验真参数决策表

## 核心思路

OCR 和 LLM 识别返回的 `invoiceType` 编码体系不统一，不同来源编码无法保证一致。
**验真参数决策不依赖任何编码，直接基于发票名称关键字匹配。**

无论数据来源（百望 OCR、LLM 多模态识别、用户口述），只要拿到发票名称就能决定验真参数。

## 发票名称关键字匹配规则

按以下顺序依次匹配，命中即停止：

### 第一步：排除数电纸质发票（优先于号码特征判断）

如果发票名称中包含 **"纸质"** 关键字，**跳过数电号码特征判断**，直接进入第二步按名称关键字匹配。

这是因为数电纸质发票（OCR 编码 85/86/87/88）虽然号码可能为 20 位，但有发票代码，且第四项参数规则不同于纯数电票。

### 第二步：判断是否为数电票（无"纸质"字样）

不满足第一步（名称不含"纸质"）时，检查号码特征：

如果同时满足以下两个条件，则判定为纯数电票：
1. 发票号码为 20 位纯数字
2. 发票代码为空或不存在

此时统一按**数电规则**处理：`totalAmount` = 价税合计，不传 `invoiceCode`。

### 第三步：按发票名称关键字分组

不满足数电特征时，按发票名称中的关键字决定验真参数：

| 组别 | 关键字匹配（名称中包含） | 验真参数 | 第四项参数 |
|------|----------------------|----------|----------|
| **专票组** | "专用发票" 或 "专用" | totalAmount + invoiceCode | 不含税金额 |
| **机动车组** | "机动车销售" | totalAmount + invoiceCode | 不含税金额 |
| **二手车组** | "二手车销售" | totalAmount + invoiceCode | 车价合计 |
| **普票组** | "普通发票" 或 "普通" 或 "通行费" 或 "卷票" 或 "卷式" | checkCode_6 | 校验码后6位（无校验码时取20位号码后6位或8位号码后6位） |

### 第四步：无法匹配 → 标记并追问

发票名称不包含上述任何关键字，且不符合数电特征时：
- 标记为 `UNKNOWN_INVOICE_TYPE`
- 跳过验真，仅做归档记录
- 向用户确认发票类型

## 验真参数决策伪代码

```
def decide_verify_params(invoice_number, invoice_code, invoice_name, amounts):
    # Step 1: 名称含"纸质"先排除纯数电规则
    if "纸质" in invoice_name:
        # 进入第二步关键字匹配
        pass
    # Step 2: 数电判断（仅当名称不含"纸质"时）
    elif len(invoice_number) == 20 and invoice_number.isdigit() and not invoice_code:
        return {
            "invoiceCode": None,  # 数电不传
            "totalAmount": amounts["价税合计"],
            "checkCode_6": None,
        }

    # Step 3: 按名称关键字匹配
    if "专用" in invoice_name:
        return {"totalAmount": amounts["不含税"], "checkCode_6": None}
    if "机动车销售" in invoice_name:
        return {"totalAmount": amounts["不含税"], "checkCode_6": None}
    if "二手车销售" in invoice_name:
        return {"totalAmount": amounts["车价合计"], "checkCode_6": None}
    if any(kw in invoice_name for kw in ["普通", "通行费", "卷票", "卷式"]):
        # 优先取校验码，无则取号码后6位
        check_code = amounts.get("校验码") or ""
        if len(check_code) >= 6:
            return {"checkCode_6": check_code[-6:], "totalAmount": None}
        else:
            # 兜底：取发票号码后6位（8位或20位号码通用）
            return {"checkCode_6": invoice_number[-6:], "totalAmount": None}

    # Step 4: 无法匹配
    return UNKNOWN
```
