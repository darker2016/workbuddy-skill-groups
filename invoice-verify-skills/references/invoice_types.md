# 发票类型与验真参数

> ⚠️ **发票名称匹配规则**（纸质排除 → 数电判断 → 关键字分组 → 兜底追问）的完整决策流程见 [ocr_to_verify_type_mapping.md](ocr_to_verify_type_mapping.md)，本文档不重复定义。

## 支持的文件格式

| 格式                         | 处理方式              | 说明             |
| -------------------------- | ----------------- | -------------- |
| .png / .jpg / .jpeg / .bmp | 上传 OSS → 百望 MCP OCR 识别 | 单张或多张图片        |
| .pdf                       | 上传 OSS → 百望 MCP 识别     | 支持多页           |
| .ofd                       | 影像识别采集 MCP（裸 base64） | 使用 `baiwang.image.invoices.recogcollect` |
| .xlsx / .xls               | 内置解析          | 读取工作表按列名匹配发票字段 |
| .csv                       | 内置解析          | 按 CSV 列名匹配发票字段 |
| .xml                       | 影像识别采集 MCP（裸 base64） | 识别失败时才按电子发票数据电文规范解析 |

## 四要素字段定义（与 API 参数名对齐）

| 字段   | API 参数名         | 格式要求                     | 示例           | 适用场景 |
| ---- | --------------- | ------------------------ | ------------ | ------ |
| 发票代码 | `invoiceCode`   | 10位或12位纯数字；数电票不传 | 044002100311 | 非数电发票 |
| 发票号码 | `invoiceNumber` | 8位纯数字（数电为20位）            | 24891456     | 全部票种 |
| 开票日期 | `billingDate`   | YYYY-MM-DD 或 YYYY年MM月DD日 | 2025-03-15   | 全部票种 |
| 校验码  | `checkCode_6`   | 6位纯数字（校验码末6位） | 123456       | 普票类（名称含"普通"/"通行费"/"卷票"/"卷式"） |
| 金额   | `totalAmount`   | 金额数字（不含逗号，保留两位小数）        | 26548.67     | 非普票类 |

## 文本输入场景（用户口述/手动输入）

用户手动提供金额时，不需要说明含税/不含税，系统按已知发票名称/票种自动匹配：

| 用户输入                  | 已知票种 | 系统处理 |
| --------------------- | ------ | ------ |
| "金额 1000 元" | 名称含"专用"/"机动车" | 视为不含税金额 → `totalAmount = "1000.00"` |
| "金额 1000 元" | 数电（20位号+无代码） | 视为价税合计 → `totalAmount = "1000.00"` |
| "金额 1000 元" | 名称含"普通"/"通行费"/"卷票" | 视为校验码（若为6位）→ `checkCode_6 = "1000"` 或追问确认 |
| "金额 1000 元" | **未知票种** | 追问发票类型后再决定 |
| "价税合计 11300" | 任意 | 数电取价税合计 = 11300，专票反推不含税 = 10000 |
| "不含税金额 10000，税额 1300" | 任意 | 提取不含税金额 10000 → `totalAmount = "10000.00"`（若名称要求不含税） |
| "车价合计 50000" | 名称含"二手车" | → `totalAmount = "50000.00"` |
| "校验码 123456" | 普票 | → `checkCode_6 = "123456"` |

**仅需追问的情况**：

- 发票名称不包含任何关键字 **且** 用户未说明发票类型
- 目标取值字段为空（如专票票面「金额」栏模糊/缺失）

## 旅客运输服务发票识别（抵扣判断用）

### XML 结构化字段映射（依据《电子发票数据电文规范 v0.36》）

| 判断条件   | XML 字段路径                                                       | 数据类型 | 说明                               |
| ------ | -------------------------------------------------------------- | ---- | -------------------------------- |
| 票种     | `Header/InherentLabel/EInvoiceType/LabelCode`                  | C..2 | `01`=电子发票（Fppz_dm取值为01、02、05、06） |
| 票种类    | `Header/InherentLabel/GeneralOrSpecialVAT/LabelCode`           | C..2 | `02`=普通发票                        |
| 开票日期   | `TaxSupervisionInfo/IssueTime`                                 | 日期时间 | 发票生成时间，取日期值                      |
| 旅客运输标识 | `EInvoiceData/SpecificInformation/PassengerTransportation`     | 节点   | 存在即表示为旅客运输服务                     |
| 出行人信息  | `EInvoiceData/SpecificInformation/PassengerTransportation` 子节点 | 0…N  | 0条或多条出行人记录                       |

### 票面关键字识别（PDF/图片场景）

| 识别目标   | 关键字/位置                        |
| ------ | ----------------------------- |
| 票种     | 发票顶部标题含「电子发票（普通发票）」           |
| 旅客运输标识 | 发票左上角二维码旁边含「旅客运输服务」字样         |
| 出行人信息  | 票面包含出行人姓名/证件号/出行日期/出发地/到达地等字段 |

## 旅客运输可抵扣税额计算

根据票据类型计算可抵扣税额：

| 票据类型           | 计算公式                               |
| -------------- | ---------------------------------- |
| 增值税电子普通发票/专用发票 | 可抵扣税额 = 票面注明的税额                    |
| 航空运输电子客票行程单    | 可抵扣税额 = (票价 + 燃油附加费) ÷ (1+9%) × 9% |
| 铁路车票           | 可抵扣税额 = 票面金额 ÷ (1+9%) × 9%         |
| 公路、水路客票        | 可抵扣税额 = 票面金额 ÷ (1+3%) × 3%         |

**抵扣前提**：票面需注明旅客身份信息（姓名、证件号），用途为生产经营相关差旅（非集体福利、个人消费），人员为本单位雇员或劳务派遣员工。
