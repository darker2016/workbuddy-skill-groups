# 中国法条与案例检索

## 概述

中国法律条文、法规、司法解释和案例的检索工具。是企业法务专家团处理中国境内法务事务的首选数据源检索工具。

## 使用场景

- 用户要求查询中国法律条文、法规、司法解释或案例
- 专家需要核验法条、案例、裁判要旨是否真实存在
- 企业尽调需要查询涉诉、行政处罚、失信、商标、专利、欠税等公开风险线索
- 中国本地化分析需要把模型判断落到可追溯来源上

## API 密钥与成本规则

- API 密钥须来自 `YD_API_KEY` 环境变量或 `scripts/.env` 文件
- 密钥缺失时不得凭空编造结果，须解释需要配置 API 密钥
- 核心法律/案例检索可在用户要求研究时直接执行
- 高成本调用（如案例详情、企业深度尽调、幻觉检测）应先确认，除非用户已要求该深度
- 每次调用后报告 API 额度消耗情况

## 路由指南

| 需求 | 使用端点 |
|------|---------|
| 法条依据 | `search`、`keyword`、`detail` |
| 案例检索 | `case`、`case-semantic`；确认相关性后再调用 `case-detail` |
| 企业风险 | 先用 `enterprise-search`，再用 `enterprise-base` / `enterprise-summary`，然后按需选 `enterprise-list` 维度 |
| 引用核验 | 仅在用户要求或法律引用将实质性影响结论时使用 `hall-detect` |

## 输出规范

返回简洁的研究发现，附带来源标签、检索时间和仍需要律师核实的内容。保持结果对相关团队成员可用，而非转储原始 API JSON。

## 与主工作流的集成

在企业法务专家团工作流中，`china-legal-research` 在以下阶段使用：

- **Workflow A Phase 2**：当 AI 供应商交易涉及中国监管要求时
- **Workflow B Phase 2**：当产品发布涉及中国行业法规时
- **Workflow C Phase 2**：当 M&A 尽调需要中国企业信息核实时
- **Workflow D Phase 2**：当用工审查涉及中国劳动法规时
- **任何涉及中国法律的独立研究请求**

## 重要规则

- 搜索结果须标注数据来源、检索时间
- 不得编造法条或判例号
- 不同数据库覆盖范围有限，标注"搜索结果基于 XX 数据库"
- 法律法规有修订版本的，注明现行版本与生效日期
