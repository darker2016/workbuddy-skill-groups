## 1. 安全总体架构

### 1.1 安全总体架构图

> **总览**：1 张图，叠加展示——网络分区、流量入口安全产品（DDoS / WAF / CLB / 防火墙）、主机/容器安全、KMS、堡垒机、安全运营中心（SOC）、日志服务（CLS）、审计中心。
>
> **说明**：下表为常见安全组件**参考清单**，实际编写时需**根据本方案实际涉及的组件**进行说明，未涉及的组件可删除，方案中引入的其他安全组件需补充。

| 组件 | 作用 |
| --- | --- |
| DDoS | 抵御流量攻击，避免业务不可访问 / 慢 |
| WAF | 防御 Web 应用攻击、CC、防篡改、DNS 劫持 |
| 云防火墙 | 南北向（互联网 / NAT 边界墙）、东西向（VPC 间）流量管控 |
| 主机安全 | 资产梳理、漏洞修补、恶意文件检测、应用防御 |
| 数据安全审计 | 数据加密、脱敏、安全审计、攻击防护 |
| KMS | 敏感数据加密（手机号、身份证等）+ AKSK 白盒保护 |
| 堡垒机 | 保护和管理企业网络的运维通道 |
| NAT 网关 | 隐藏内部网络架构、SNAT / DNAT |
| SOC（安全运营中心） | 资产管理、资产体检、端口/弱口令/云配置扫描 |

### 1.2 威胁模型

> **总览**：采用 **STRIDE** 或 **攻击树** 方法，识别并规避系统可能存在的威胁风险。
> 每条威胁必须对应到第 2~10 章的某项缓解措施，形成"**威胁—缓解措施映射表**"。

| 威胁类别（STRIDE） | 典型示例 |
| --- | --- |
| 仿冒（Spoofing） | 身份伪造、AKSK 泄露、token 重放 |
| 篡改（Tampering） | 请求参数、数据库、配置文件 |
| 否认（Repudiation） | 缺少操作审计 |
| 信息泄露（Information Disclosure） | 敏感数据、密钥、日志 |
| 拒绝服务（DoS） | DDoS、CC、慢查询 |
| 权限提升（Elevation of Privilege） | 水平 / 垂直越权、容器逃逸 |

## 2. 身份与访问管理（IAM）

### 2.1 身份认证（Authentication）

#### 2.1.1 用户登录

- **多种登录方式**：账号密码、手机号验证码、邮箱验证码、第三方互联账号（OAuth 2.0 / OIDC / SAML / CAS）、扫码登录
- **图形验证强制场景**：登录、获取手机号 / 邮箱验证码
- **MFA / 2FA**：管理端、敏感操作、控制台子账号必须开启 MFA 等机制（密码 + OTP / 硬件令牌 / 手机验证码）

#### 2.1.2 密码策略（量化阈值）

> **说明**：下表为常见密码策略**参考基线**，实际编写时需**根据本方案的业务场景、用户类型、合规要求**确认所需的策略项及阈值，未涉及的策略项可删除，特殊场景的额外策略需补充。

| 项 | 基线值 |
| --- | --- |
| 最小长度 | ≥ 8 位（业务系统）；≥ 12 位（云控制台、CVM、数据库） |
| 复杂度 | 同时包含数字、大写字母、小写字母、特殊字符（可配置开关） |
| 加密传输 | 前端 RSA 加密提交，密文传输 |
| 加密存储 | bcrypt / scrypt / argon2 加盐哈希；**禁止 MD5 / SHA1 明文** |
| 更换周期 | 默认 90 天提醒（可配置） |
| 重复限制 | 不允许使用最近 5 次的密码 |
| 失败锁定 | 连续失败 5 次锁定 1 小时（次数和时长可配置） |
| 自动登出 | 30 分钟无操作自动登出 |
| 注册策略 | 不允许用户自行注册，仅管理员创建；改密需原密码验证 |

#### 2.1.3 会话管理

- **Token 类型**：JWT / Session ID，**禁止 URL 携带 token**
- **有效期**：默认有效期 24 小时（可配置），提供续期与主动注销接口
- **互踢机制**：管理端同账号仅允许单设备登录，新登录踢掉旧会话，异地登录需提示
- **风险登录**：异地 / 异常登录二次验证

### 2.2 授权与权限控制（Authorization）

#### 2.2.1 权限模型

默认采用 **RBAC0**（用户—角色—权限），权限粒度敏感的场景叠加 **ABAC**（属性条件）。

#### 2.2.2 权限维度

- **接口级**：API 网关统一鉴权 + 接口签名（HMAC-SHA256）
- **数据级**：行级（按归属人 / 租户）、列级（字段脱敏）
- **功能级**：菜单、按钮、操作的细粒度控制

#### 2.2.3 多租户隔离

明确选择物理隔离 / 逻辑隔离，**私有化部署默认物理隔离**。

#### 2.2.4 越权防护

水平越权（IDOR）、垂直越权，每个查询 / 修改接口必须校验资源归属。

### 2.3 云账号与云资源访问

#### 2.3.1 账号策略

使用子账号访问腾讯云，**禁止直接使用主账号**，禁止为主账号创建访问密钥。

#### 2.3.2 权限分离

人员权限（密码 + MFA，控制台 / 堡垒机使用）与服务权限（AKSK / Token，应用调用使用）分离。

#### 2.3.3 高敏权限分离

高敏操作账号权限与低敏操作隔离。

## 3. 数据安全

### 3.1 数据分级

| 等级 | 类别 | 示例 | 处理策略 |
| --- | --- | --- | --- |
| L1 | 公开 | 产品介绍、公开 API 文档 | 无特殊管控要求 |
| L2 | 内部 | 业务配置、内部流程 | 需内部鉴权访问 |
| L3 | 敏感 | 客户姓名、订单、通话记录 | 加密传输、加密存储、脱敏展示，必要时审计 |
| L4 | 高敏 | 手机号、身份证、银行卡、密码、AKSK | 基于 KMS 加密、强脱敏，严格审计 |

### 3.2 数据传输安全

#### 3.2.1 公网访问

全链路 "HTTPS / TLS 1.2+"，禁用 SSLv3、TLS 1.0/1.1。

#### 3.2.2 服务内部访问

- 低敏感、腾讯内部访问：HTTP 协议寻址访问
- 高敏感、跨租户访问：mTLS 双向证书认证，认证服务访问身份

#### 3.2.3 数据库连接

- 低敏感、腾讯内部访问：可明文数据传输
- 高敏感、跨租户访问：SSL（MySQL、MongoDB 内置功能）

#### 3.2.4 接口签名

> **说明**：以下为常见接口签名方案**参考示例**，实际编写时需**根据本方案的接口类型、调用方、安全等级**设计具体的签名机制（算法、签名字段、防重放策略等）。

参考示例：HMAC-SHA256 防篡改 + nonce + timestamp 防重放。

#### 3.2.5 传参禁忌

> **说明**：以下为常见传参禁忌**参考示例**，实际编写时需**根据本方案的接口形式、参数类型、敏感数据范围**梳理具体的传参约束清单。

参考示例：GET 传递敏感参数；**禁止 URL 携带 token、AKSK、手机号等敏感信息**。

### 3.3 数据存储安全

#### 3.3.1 加密存储（仅敏感数据需要）

> **说明**：以下为常见加密存储方案**参考示例**，实际编写时需**根据本方案的数据敏感度、数据体量、性能要求**确认所采用的加密算法、加密层级及密钥管理方式，未涉及的可删除，特殊场景的额外方案需补充。

- **对称加密**：AES-256（业务侧大数据量）
- **非对称加密**：RSA（密钥交换、签名）
- **落盘加密**：数据库、对象存储（COS）启用服务端加密（SSE）；高敏数据使用客户端加密（CSE）
- **密钥管理**：KMS 托管，**禁止硬编码到代码或配置文件**

#### 3.3.2 敏感字段单向哈希加盐

密码、token 摘要等不可逆字段。

#### 3.3.3 数据库安全

> **说明**：系统涉及数据库操作，进行如下内容设计。

- 账号最小权限；**禁止业务直连 root**；运维使用单独账号
- 开启慢查询日志、审计日志；日志独立存储到 CLS / COS
- 启用 SSL 连接、备份加密

#### 3.3.4 文件存储安全

> **说明**：业务涉及敏感文件数据存储、外链、分享时，考虑如下安全设计。

- 启用服务端加密（SSE）
- 私有桶 + 预签名 URL（短有效期）
- 防盗链（Referer 校验）
- 上传文件病毒扫描 / 类型校验（魔数 + 扩展名 + 大小）

#### 3.3.5 数据使用安全

- **展示脱敏**：手机号 `138****8888`；身份证 `110***********1234`；银行卡 `**** **** **** 1234`
- **数据导出管控**：导出审批流 + 水印（操作人 + 时间）+ 行数限制 + 操作审计
- **数据销毁**：逻辑删除 + 定期物理销毁；明确过期数据清理策略

#### 3.3.6 数据备份与异地容灾

| 存储类型 | 备份策略 |
| --- | --- |
| MySQL / MongoDB / PG / ES | 周期性备份，跨地域备份至 COS |
| Redis | RDB + AOF，按 RPO 选择性异地 |
| COS 重要数据 | 版本控制开启，跨地域复制 |

#### 3.3.7 隐私合规

> **说明**：系统涉及"合规"问题时，专项进行合规评审。

- 个人信息保护（GDPR、《个人信息保护法》、《数据安全法》）
- 用户同意机制（隐私政策 + 用户明示同意）；数据最小化采集；可被遗忘权（数据删除请求处理流程）
- 数据出境合规评估（涉及跨境业务时必填）

## 4. 应用安全

> **说明**：本章涉及的输入/输出防护、业务逻辑安全、第三方与供应链安全等内容均为**参考示例**，实际编写时需**根据本方案的应用类型（Web / API / 移动端）、对外暴露面、业务关键操作**梳理具体的防护清单，未涉及的可删除，特殊场景（如对外开放 API、支付类业务等）的额外防护需补充。

### 4.1 输入安全

> **说明**：以 OWASP Top 10 为典型风险场景，结合腾讯安全规约。

| 风险 | 防护措施 |
| --- | --- |
| SQL 注入 | 强制参数化查询（PreparedStatement / ORM 参数绑定），动态字段（order by / group by）使用模版校验 |
| XSS | 输出编码（HTML / JS / URL）、CSP 策略、Cookie HttpOnly + Secure |
| CSRF | CSRF Token、SameSite Cookie、关键操作二次验证 |
| 命令注入 | 系统禁用动态命令执行，如需调用使用白名单参数化数组形式 |
| 反序列化 | 禁用危险反序列化（fastjson autoType、Jackson @class） |
| SSRF | URL 白名单、内网 IP 黑名单 |
| 文件上传 | 类型校验、扩展名白名单、大小限制、随机重命名、隔离存储桶 |
| XXE | 关闭 XML 外部实体解析（DTD） |
| 开放重定向 | 跳转 URL 白名单 |

### 4.2 输出安全

#### 4.2.1 统一异常处理

错误码及信息提示，**禁止暴露堆栈、SQL、内部路径**。

#### 4.2.2 调试信息

调试信息（debug、stacktrace）严禁出现在生产环境响应中。

### 4.3 业务逻辑安全

#### 4.3.1 越权防护

水平越权（IDOR）每个接口必检；垂直越权由 RBAC + 接口拦截器双层校验。

#### 4.3.2 防刷防爬

限流（IP / 用户 / 接口三维度）、滑动验证码、设备指纹、人机识别。

#### 4.3.3 防薅羊毛

风控规则、黑名单、行为分析（结合 SOC 告警）。

#### 4.3.4 幂等性

防重复提交（前端 token + 后端唯一键）。

#### 4.3.5 关键业务二次验证

转账、删除、批量导出、密钥变更等需短信 / MFA 二次确认。

### 4.4 第三方与供应链安全

> 针对系统需引入的第三方组件进行安全性评估。

- 第三方组件漏洞扫描（公司组件安全扫描），CICD 强制门禁
- 开源协议合规审查（GPL / AGPL 商业风险）
- 第三方服务接入安全评估（数据传输、存储位置、合规资质）
- 供应商安全管理：合同含数据保护与审计条款；定期复评
- 镜像源、Maven 仓库、NPM 仓库使用"内网镜像 + 白名单"，避免被投毒

## 5. 网络与基础设施安全

> **说明**：本章涉及的网络分区、流量管控、主机/容器/中间件安全等内容均为**参考示例**，实际编写时需**根据本方案的部署形态（公有云 / 私有化 / 混合云）、网络拓扑、对外暴露需求、运维通道**确认具体方案，未涉及的可删除，特殊场景（如多 Region 部署、专线接入、堡垒机方案等）的额外内容需补充。

### 5.1 网络分区

- 区分 **DMZ VPC**（对外暴露）、**业务 VPC**（int / uat / prod 隔离）、**DevOps VPC**（CICD / 运维），避免网络渗透，以及内部研发阶段外泄
- VPC CIDR 不重叠；容器 CIDR 与 VPC CIDR 不冲突
- 限制生产 VPC 与公网的访问通路和方式，通常只能通过 NAT 网关出公网、CLB 等网络组件出入公网

### 5.2 流量管控（南北向 + 东西向）

#### 5.2.1 公网入口防护

- DDoS 高防：流量清洗，业务侧不感知
- WAF：访问控制，黑白名单，攻击日志，请求清洗
- CLB / Ingress：仅暴露 443（HTTPS），80 强制跳 443

#### 5.2.2 网络边界防护

- 云防火墙：互联网边界墙 + NAT 边界墙 + 入侵防御开启，按需开启 VPC 间防火墙

#### 5.2.3 主机流量防护

- 安全组 + ACL：默认拒绝 + 显式放通，**禁止 0.0.0.0/0 开放 22/23/3306/6379 等敏感端口**

#### 5.2.4 运维通道防护

- 堡垒机：运维日志、文件传输日志、数据库操作日志全量记录

### 5.3 主机与容器安全

#### 5.3.1 主机安全

- 最小化安装、关闭非必要服务、定期补丁
- 账号管控：**严禁 root 远程登录**，运维使用独立账号
- 登录密码：**优先 SSH 密钥登录（密钥对）**，密码登录仅作兜底
- 安装主机安全组件：根据公司安全要求安装洋葱等组件（漏洞扫描、木马查杀、密码破解阻断、漏洞防御、漏洞自动修复）

#### 5.3.2 容器 / K8s 安全

- K8s RBAC 最小化、NetworkPolicy 隔离命名空间、Pod 安全策略
- 容器镜像扫描（漏洞、敏感信息）、本地 + 仓库镜像风险扫描
- 异常进程监控、容器逃逸检测、K8s API 异常请求监控
- **不在镜像中打包密钥**；通过 KMS / 配置中心运行时下发
- 禁止特权容器、高权限容器，如需特殊修改容器权限需谨慎配置

### 5.4 中间件安全

#### 5.4.1 网络隔离

仅业务 VPC 内的应用层可访问；**禁止公网暴露**。

#### 5.4.2 强制鉴权

数据库、Redis、MQ 等中间件强制鉴权，修改默认账号，开启 SSL。

#### 5.4.3 审计

启用审计日志，关键中间件接入数据安全审计（DBAudit）。

## 6. 密钥与凭证管理

> **说明**：本章涉及的密钥分级、AKSK 泄露防护方案、密钥使用红线等内容均为**参考示例**，实际编写时需**根据本方案涉及的密钥类型、云服务商能力、合规要求**确认具体的存储/管理/轮换方式，未涉及的可删除，特殊场景（如硬件 HSM、自建 Vault、跨云密钥同步等）的额外方案需补充。

### 6.1 密钥分级与存储

| 密钥类型 | 存储 / 管理方式 |
| --- | --- |
| 数据库密码、第三方 AppKey | 配置中心加密 / SSM 凭据 |
| 云 AKSK（COS、MQ 等） | KMS 白盒密钥 + SSM 凭据动态获取 |
| 服务间通信密钥 | KMS / Vault |
| 用户密码 | bcrypt 加盐哈希（不可逆） |
| API Key / Token | 分级授权、可撤销、有限期 |

### 6.2 AKSK 泄露防护

#### 6.2.1 方案 A：KMS 白盒密钥

使用 KMS 白盒密钥功能加密 SecretId / SecretKey；业务模块通过 KMS SDK 解密拿到 AKSK 明文，再调用 SSM 接口获取凭据内容。

#### 6.2.2 方案 B：CAM 策略限制

通过腾讯云 CAM 权限策略，绑定资源访问权限，规避 AKSK 访问机制。

### 6.3 密钥使用红线

- **严禁** AKSK / 数据库密码 / Token 出现在源码、Git 仓库、前端代码、日志、错误堆栈、URL、配置明文
- **禁止**多业务共用同一对 AKSK；按业务 / 环境隔离签发
- 上线前必须通过代码扫描（可联动 CICD 流水线红线拦截）+ 镜像扫描，识别硬编码密钥
- AKSK 一旦疑似泄露，立即在 CAM 控制台禁用并轮换；事后排查访问日志确认影响面
- 每一对 AKSK 明确所有人、用途、存储位置、轮换日期，**上线 checklist 中包含"密钥扫描通过"项**

## 7. 运行时安全：监控、审计、应急

### 7.1 安全事件监控

- **入侵检测（IDS / IPS）**：主机安全告警、WAF 攻击日志、SOC 告警接入
- **敏感操作**：数据导出、批量删除、密钥变更、用户角色变更
- **登录异常**：异地登录、连续失败、夜间登录、非常用设备
- **权限异常**：越权请求、敏感接口高频调用、非常规权限提升

### 7.2 审计日志

- **云资源审计日志**：事件名称、区域、时间、事件源、源 IP、请求 ID、操作者
- **业务操作审计**：操作时间、来源应用、来源类型、来源 IP、操作者、事件名称、读写类型、扩展字段
- **数据库审计**：SQL、执行人、客户端 IP、时间、影响行数
- **堡垒机日志**：主机 / 数据库运维日志、文件传输日志
- **WAF / 防火墙日志**：攻击日志、访问日志、拦截记录

### 7.3 应急响应

- **应急预案**：漏洞响应、数据泄露、被攻击、密钥泄露、勒索病毒
- **备份与恢复**：明确 RPO / RTO；**每年 ≥ 1 次灾备演练**
