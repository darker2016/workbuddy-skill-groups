### 3.3 高可用与故障容错

#### 3.3.1 故障域划分

| 故障域 | 影响范围 | 容错机制 | 预期 RTO |
| --- | --- | --- | --- |
| 单 Pod | 单实例 | K8s 自动重建 | < 30s（瞬时无影响） |
| 单 Node | 该 Node 上的全部 Pod | K8s 调度到其他 Node | < 1min |
| 单 AZ | 该 AZ 全部资源 | 跨 AZ 部署，自动切换 | ≤ 5min |
| 单 Region | 全部主资源 | 灾备 Region 切换 | < 1h |

> **可用性来源拆分**：哪部分由云厂商 SLA 兜底（IaaS / PaaS）、哪部分由本系统自身保障（业务逻辑容错 / 重试 / 降级）。

#### 3.3.2 负载均衡

4 层 / 7 层；健康检查策略（路径 / 频率 / 失败阈值）。

#### 3.3.3 健康检查配置

> 健康检查路径、频率、失败阈值、连续成功阈值等。

#### 3.3.4 弹性伸缩配置

触发指标（CPU / QPS / 队列长度）+ 阈值 + 伸缩区间（min / max 副本数）。

#### 3.3.5 容灾切换配置

### 4.2 流水线（CI / CD）

| 阶段 | 触发条件 | 关键动作 | 失败处理 |
| --- | --- | --- | --- |
| 构建 | 自动 / 手动触发 | 拉代码 → 编译 → 单元测试 → 构建产物 | 阻断后续阶段 |
| 代码扫描 | 构建后 | 静态扫描（SonarQube 等）/ 代码规范 / 复杂度 | 不达标阻断 |
| 安全扫描 | 构建后 | SCA（依赖漏洞）/ SAST（代码漏洞）/ 镜像漏洞扫描 | 高危阻断 |
| 集成测试 | 合并到集成分支 | 集成测试 / 接口测试 / 契约测试 | 阻断后续阶段 |
| 制品打包构建 | 构建/测试通过 | 构建镜像 / 制品包，推送到仓库并打 Tag | 失败重试 |
| 部署集成环境 | Package 完成 | 自动部署到 dev / int 环境 | 失败回滚到上一版本 |
| 部署测试环境 | 手动触发 | 部署到预发；冒烟测试 | 失败回滚到上一版本 |
| 部署生产环境 | 人工审批 | 按发布策略灰度上线 | 按回滚策略实施回滚 |

#### 8.1.2 资源容量推算

> 根据系统实际涉及的资源类型逐项填写，每项需给出推算公式与当前 / 目标阶段配置。
> 常见推算参考：
> - **应用实例数** = 峰值 QPS / 单实例承载 QPS × 冗余系数（建议 1.3 ~ 1.5）
> - **带宽** = 峰值 QPS × 单次平均报文大小 × 冗余系数
> - **MySQL** = 写 TPS × 平均 SQL 影响行数 → IOPS；读 QPS / 单从库承载 → 从库数
> - **Redis** = 单 Key 平均大小 × Key 数量 × 冗余系数
> - **MQ（Kafka / TDMQ）** = 峰值 TPS / 单分区承载 → 分区数
> - **对象存储 / 文件存储** = 日增量 × 保留天数 × 副本系数

#### 8.1.3 扩容触发与路径

> 根据系统实际涉及的资源类型逐项填写扩容策略。
> 常见参考维度：
> - **触发指标**：CPU / 内存 / IOPS / QPS / 队列堆积 / 存储水位等
> - **扩容方式**：水平扩容（HPA / 加副本）/ 垂直扩容（升配）/ 架构演进（读写分离、分库分表、集群扩节点）
> - **扩容耗时**：明确从触发到生效的预期时间，区分自动 vs 人工

#### 8.1.4 容量水位线

> 容量水位采用四级分层模型，每级对应固定的处置动作；具体阈值需根据资源类型与业务弹性需要逐项评估填写。

**水位分层定义（标准）**

| 水位 | 含义 | 动作 |
| --- | --- | --- |
| 健康水位 | 正常运行区间 | 无动作 |
| 预警水位 | 接近瓶颈，需关注 | 告警，启动扩容评审 |
| 危险水位 | 即将超载 | 自动扩容（如支持）或紧急扩容 |
| 红线水位 | 必须保护 | 触发限流 / 降级 |
